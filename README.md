# 康纳同学 Connor

康纳同学是一款本地优先的 macOS Agent 助手。它把对话、文件、浏览器、数据源、技能、任务和长期记忆放进同一个原生工作空间，让写作、研究、开发、资料整理与日常事务可以持续推进，而不是停留在一次问答。

康纳同学可以在你授权后读取资料、操作工作目录、访问网页和调用本地数据源。文件修改、命令执行、邮件发送等重要操作仍受权限、审批与审计流程约束。

## 主要能力

- **会话与长期协作**：按会话保留上下文、工作目录、附件、浏览器状态和模型选择，并使用后台 Memory OS 整理有来源的近期信息、知识、偏好与关系。
- **人格定制**：用自然语言定义康纳同学的性别自我呈现、核心特征、沟通与思考方式、主动性、情绪基调和行为边界，让这些偏好持续体现在后续对话中。
- **文件与代码工作区**：读取、创建和修改本地文件，运行构建、测试与诊断命令；可在会话中浏览目录树、查看 Git 状态并预览文本、代码、图片、PDF 和本地 HTML。
- **研究与内置浏览器**：搜索和读取网页，管理标签页、历史、书签、下载与站点权限，也可让 Agent 在授权范围内操作页面和辅助填写表单；使用网页资料时会附上实际采用的参考链接。
- **笔记与附件**：新建或批量导入笔记，处理文本、Markdown、代码、日志、JSON、CSV、图片以及常见 PDF、Office 和 iWork 文档，并把结果沉淀回会话。
- **原生数据源**：连接邮件、日历、RSS、联系人与人际关系数据；搜索和整理信息、管理人物资料，或在审批后执行日历和邮件操作。
- **技能、任务与自动化**：安装和管理技能，也可从 Claude Code、Codex、Cursor、Copilot、Gemini CLI、OpenCode、Windsurf、Cline 等常见 Agent 目录导入技能；支持定时任务和由会话状态触发的任务。
- **语音与生成媒体**：支持会话内语音转文字；模型能力允许时，可生成图片或音频，并作为本地附件保存和回放。
- **全局搜索与组织**：搜索会话和本地内容，使用状态与标签组织工作，并从搜索结果直接开始新的会话。

## 开始使用

### 1. 安装应用

康纳同学支持 macOS 14 及以上版本。请前往 [GitHub Releases](https://github.com/duanshiwen/connor-graph-agent-mac/releases) 获取最新安装包，并根据 Mac 的处理器选择文件：

- Apple M 系列芯片：选择文件名包含 `arm64` 的 DMG。
- Intel 芯片：选择文件名包含 `x86_64` 的 DMG。

打开 DMG，将“康纳同学”拖入“Applications”文件夹。

当前公开安装包使用 Ad-hoc 签名，没有经过 Apple 公证。首次启动时：

1. 在“应用程序”文件夹中打开“康纳同学”。如果 macOS 提示 Apple 无法验证，请点击“完成”。
2. 打开“系统设置 → 隐私与安全性”，向下滚动到“安全性”。
3. 在已阻止使用“康纳同学”的提示旁点击“仍要打开”，按系统要求完成身份验证。
4. 警告再次出现时点击“打开”。

完成一次确认后，后续可以正常双击运行，不需要执行终端命令。“仍要打开”只会在尝试启动应用后出现；受组织管理的 Mac 可能不允许用户自行放行。

### 2. 连接模型

打开“设置 → AI”，添加并验证模型连接。康纳同学目前提供以下几类入口：

- OpenAI Responses API、Anthropic / Claude API。
- ChatGPT Plus 的 Codex 登录，以及 GitHub Copilot 授权。
- DeepSeek、Xiaomi MiMo、Qwen、豆包、Kimi、GLM、MiniMax、阶跃星辰等常用国内模型服务。
- Google AI Studio、OpenRouter、Groq、Mistral、xAI、Cerebras、Hugging Face 等兼容服务。
- Ollama 等本地模型，以及自定义 OpenAI / Anthropic 兼容接口。

部分服务可直接选择 Coding Plan 或 Token Plan，并使用对应的专属 Endpoint 和套餐凭证。不同连接支持的工具调用、视觉、图片或音频能力可能不同，应用会在添加连接时进行验证。

API Key 和运行凭证会保存在本机加密凭证库中，不会以明文写入项目文件、会话内容或普通设置文件。

### 3. 创建第一个会话

新建会话后，直接描述希望完成的结果。例如：

```text
帮我整理这份资料，提炼结论并列出下一步行动。
```

```text
检查这个项目为什么构建失败，修复后运行相关测试。
```

```text
研究这个问题，核对最新资料，并在结论后列出参考链接。
```

```text
查看我接下来三天的安排，找出两个适合集中工作的时间段。
```

处理本地项目时，请先为会话选择工作目录。需要邮件、日历、RSS、联系人或 MCP 工具时，再按需连接对应数据源并授予必要权限。

## 会话、笔记与技能

- **新建会话**适合问答、研究、开发、资料处理和自动化任务。
- **新建或导入笔记**适合 Markdown、长文本和需要持续沉淀的资料；批量导入时可以查看校验、去重和 AI 处理进度。
- **技能**为特定任务提供专门的指令与工作流程。输入 `/` 或使用技能入口即可选择，也可以从本机已有 Agent 工具导入。
- **状态与标签**用于组织会话；状态变化还可以触发自动化任务。

你不需要学习固定指令。说明目标、限制、可用资料和期望输出即可；如果需求还不明确，也可以让康纳同学先提出澄清问题。

## 人格定制

打开“设置 → 偏好 → 康纳同学的性格”，用自然语言描述你希望的相处方式，例如“冷静、直接，遇到风险主动提醒我，但不要替我做决定”。点击“AI 分析”后，康纳同学会生成一份结构化预览，包含性别自我呈现、总体人格、核心特征、沟通方式、思考方式、主动性、情绪基调和行为边界；确认内容后再点击“应用性格”。你也可以随时重新生成或恢复默认性格。

也可以直接在会话中提出明确的长期要求，例如“以后回答更简洁一些”或“今后发现我的方案有明显风险时请主动指出”。这类要求会更新持久人格；“这次简短一点”等仅针对当前任务的表达不会保存为长期设置。

人格会影响措辞、节奏、详略、温度和主动程度，但不会改变事实准确性、安全规则、权限边界或当前任务要求。“康纳同学”是固定名称，不属于可修改的人格字段。

## 权限与安全

康纳同学只在相关功能需要时请求系统权限，例如联系人、日历、麦克风、语音识别、位置、通知、摄像头或文件访问。macOS 的权限提示与首次启动的 Gatekeeper 确认是两套独立机制。

每个会话可以设置主要工作目录和额外允许目录。本地文件与 Shell 工具、外部网络、数据源写入和邮件发送都要经过相应策略检查；高风险操作会显示待审批内容，未经批准不会执行。建议只授权当前任务所需的目录、数据源和能力。

## 本地数据与隐私

会话、设置、任务、附件、数据源缓存、搜索索引和运行日志主要保存在本机。Memory OS 在后台分层保存原始来源、近期操作信息、可复用知识和稳定实体关系，使后续会话能够延续上下文并追溯证据；它不是需要用户手动维护的图谱编辑器。用户消息进入 Memory OS 时，L0 完整保留原文，L1 另外保存一份由模型生成并经本地校验的“历史用户意图”描述。后续近期上下文只返回这份描述，避免把过去的指令误当成当前指令；后台知识提取仍从 L0 读取完整原文，不使用 200 字预览或截断。

模型请求仍会按照你选择的连接发送给对应的模型提供方。网页搜索、邮件、日历和其他外部数据是否离开本机，取决于所选连接、已启用的数据源和具体任务。请根据自己的数据要求选择服务并控制授权范围。

## 开发

项目使用 Swift 6 和 SwiftUI，支持 macOS 14 及以上版本；全文搜索由仓库内的 Rust / Tantivy Search Kernel 提供。常用命令：

```bash
swift build
swift test
swift run connor --help
```

完整架构、模块边界、Memory OS、Search Kernel、开发命令和打包说明请参阅 [ENGINEERING.md](ENGINEERING.md)。Memory OS 的独立说明见 [MemoryOS_Architecture_and_Usage.md](MemoryOS_Architecture_and_Usage.md)。
